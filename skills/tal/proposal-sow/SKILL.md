# הצעת מחיר / SOW — הנוהל

מסמך התקשרות ברור מונע מחלוקות ו-scope creep. הנוהל בונה טיוטה מובנית שמגנה עליך ועל הלקוח. **טיוטה — לבדיקת איש מקצוע במידת הצורך.**

## שלב 1 — קלט
קבל: מי הלקוח, מה ביקש, מה כלול ומה לא, המחיר ומבנה התשלום, ולוחות הזמנים.

## שלב 2 — מבנה המסמך
1. **היקף העבודה** — מה בדיוק תספק.
2. **החרגות** — מה לא כלול (החלק הכי חשוב למניעת ויכוחים).
3. **תוצרים ואבני דרך**.
4. **מחיר ותנאי תשלום** — סכום, פריסה, מועדים.
5. **לוח זמנים**.
6. **תנאים כלליים** — שינויים, ביטול, קניין רוחני.

## שלב 3 — סימון פערים
כל פרט שלא נמסר = `[להשלים]`. אל תמציא מחיר/תנאי.

## שלב 4 — פלט
1. שמור ל-`<לקוח>_הצעה.md`.
2. בצ'אט: הצג את המסמך ואת הפערים.

## כללים
- אל תמציא מחיר/היקף — `[להשלים]`.
- ההחרגות חשובות כמו ההיקף.
- טיוטה לאישור שלך; סעיפים משפטיים — `[לבדוק עם איש מקצוע]`.
