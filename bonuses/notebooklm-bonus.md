# מדריך בונוס · חיבור NotebookLM ליצירת מצגות וסיכומים

> **חלק מחבילת הבונוסים של קורס Claude Desktop**
> מדריך צעד-צעד לשילוב בין Claude Code לבין NotebookLM של Google.

---

## מה זה NotebookLM

NotebookLM הוא כלי AI של Google שעובד כ"עוזר מחקר אישי" על בסיס מקורות שאתה מספק. בניגוד ל-ChatGPT שיודע הכל אבל לא יודע כלום על המסמכים שלך, NotebookLM קורא רק את הקבצים שהכנסת לו ועונה בהתבסס עליהם בלבד.

**למה זה שימושי לנו:**
- Claude Code מייצר תוכן מצוין: קוד, מבנה, טקסט, ניתוחים
- NotebookLM לוקח את התוכן הזה ומפיק מצגות, סיכומים, פודקאסט-style audio, ושאלות נפוצות — בלי שתצטרך לעצב כלום

---

## זרימת העבודה המומלצת

```
Claude Code                NotebookLM
-----------                ----------
יוצר תוכן ומבנה    →    מעכל ומארגן מחדש    →    מצגת / סיכום / אודיו
```

**דוגמה מהחיים:** ביצרנו את מצגת הסדנה הזאת בדיוק ככה — Claude Code הפיק את המבנה והתוכן, NotebookLM הפיק ממנו גרסאות שונות.

---

## שלב 1 · כניסה ל-NotebookLM

1. עבור לכתובת: [קישור] (notebooklm.google.com)
2. היכנס עם חשבון Google
3. לחץ על "New Notebook" (מחברת חדשה)
4. תן לה שם ברור — לדוגמה: "סדנת Claude Code — חומרים"

---

## שלב 2 · הכנסת מקורות

NotebookLM תומך בסוגי מקורות הבאים:

| סוג | איך מכניסים |
|---|---|
| קובץ PDF | גרור לתיבת המקורות או לחץ "Upload" |
| קובץ Google Doc | הדבק קישור ל-Doc |
| קובץ טקסט (.txt) | גרור או Upload |
| אתר אינטרנט | הדבק URL |
| טקסט חופשי | לחץ "Paste text" ו-הדבק ישירות |

**מה להכניס מהסדנה:**
- את הפלט של Claude Code (קוד מוסבר, מסמכי תכנון, CLAUDE.md)
- את ה-Prompt Pack (המסמך הזה ושאר החומרים)
- כל מסמך שיצרת ב-Claude Code שאתה רוצה לסכם או להפוך למצגת

**טיפ:** ניתן להכניס עד 50 מקורות לכל מחברת. כדאי להפריד נושאים — מחברת אחת לסדנה, מחברת נפרדת לפרויקט ספציפי.

---

## שלב 3 · שיחה עם המקורות

לאחר שהמקורות נטענו (עד דקה-שתיים), תוכל לשוחח עם NotebookLM:

**שאלות לדוגמה:**
- "מה הנושאים העיקריים במסמכים האלה?"
- "תסכם את הפרויקט בשלושה משפטים"
- "מה ההבדל בין X ל-Y לפי המסמכים?"
- "אילו שאלות נפוצות עשויות להיות לקורא?"

NotebookLM יענה ויצרף ציטוט מדויק מהמקור — כך שאתה תמיד יודע מאיפה כל פיסת מידע באה.

---

## שלב 4 · הפקת מצגת

NotebookLM יכול ליצור מצגת מהמקורות שלך:

1. בתפריט הצד (Notebook Guide) לחץ על **"Study guide"** או **"Briefing doc"**
2. בחר **"Presentation"** אם האפשרות זמינה (זמינות לפי גרסה)
3. לחלופין — בקש ישירות בצ'אט:

```
צור מצגת של 8–10 שקופיות על [נושא] מהמסמכים שהעלתי.
לכל שקופית: כותרת + 3 נקודות עיקריות.
```

4. NotebookLM יפיק טיוטת תוכן. אפשר אז:
   - להעתיק לתוך Google Slides / Canva / PowerPoint
   - לשלוח ל-Claude Code לעיצוב HTML (ראה שלב 6)

---

## שלב 5 · הפקת סיכום ושאלות נפוצות

מה עוד NotebookLM יכול לעשות עם המקורות שלך:

**Audio Overview (סיכום אודיו):**
- לחץ על "Audio Overview" ב-Notebook Guide
- NotebookLM יפיק שיחה טבעית בין שני "מגישים" שמסכמים את המקורות
- שימושי להאזנה בדרך, לסיכום מהיר, לתוכן פודקאסט

**FAQ:**
- בקש: "צור רשימת שאלות ותשובות נפוצות מהמסמכים האלה"
- שימושי לדף FAQ באתר, להכנה לסדנה, לתמיכת לקוחות

**Briefing Doc:**
- סיכום קצר ומובנה לכל המקורות — אידיאלי לפני פגישה

## שלב 6 · זרימת עבודה מלאה עם Claude Code

זו הדרך שבה משלבים את שני הכלים יחד:

### שלב א — Claude Code מייצר תוכן

כתוב ב-Claude Code:

```
כתוב לי מסמך תוכן מפורט עבור מצגת על [נושא].
כלול:
- 8 נושאי שקופית עם כותרות
- לכל שקופית: 3–5 נקודות תוכן
- הצעות לדוגמאות ויזואליות

פורמט: markdown נקי שניתן להעתיק.
```

### שלב ב — מכניסים ל-NotebookLM

- מעתיקים את הפלט לקובץ טקסט או מדביקים ישירות ב-"Paste text"
- מוסיפים מקורות רלוונטיים נוספים (מחקרים, מאמרים, הנחיות)

### שלב ג — NotebookLM מעשיר ומארגן

- מבקשים סיכום, שאלות, Audio Overview
- מבקשים לזהות פערים: "מה חסר במצגת לפי המקורות?"
- מבקשים עיבוד מחדש: "ארגן את התוכן לפי AIDA (Attention, Interest, Desire, Action)"

### שלב ד — חוזרים ל-Claude Code לעיצוב

מביאים את הפלט של NotebookLM בחזרה ל-Claude Code:

```
זה מבנה המצגת שיצרתי. עכשיו בנה אותה כ-HTML deck ב-reveal.js.
[מדביקים את התוכן]

דרישות עיצוב:
- dark theme
- עברית RTL
- פונטים: Polin/Assistant
- אנימציות כניסה עדינות
```

## טיפים מהשטח

**מחברות נפרדות לנושאים נפרדים.**
אל תערבב פרויקטים שונים באותה מחברת — NotebookLM עובד הכי טוב כשהמקורות קוהרנטיים.

**עדכן מקורות בחזרה.**
אם Claude Code שינה קובץ — החלף את הגרסה הישנה ב-NotebookLM. הוא יחזיר תשובות מדויקות יותר.

**בקש ציטוטים.**
NotebookLM תמיד יסמן מאיפה המידע בא. אם ראית תשובה מעניינת — לחץ על הציטוט כדי לקרוא את ההקשר המלא.

**שמור prompt-ים שעבדו.**
כשמצאת ניסוח שנתן לך מצגת טובה — שמור אותו. שימוש חוזר = חיסכון זמן.
