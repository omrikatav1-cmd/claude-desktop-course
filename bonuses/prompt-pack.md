# Prompt Pack — פרומפטים מוכנים לפי משימות

> **חלק מחבילת הבונוסים של קורס Claude Desktop**
> כל הפרומפטים נכתבו לשימוש ישיר ב-Claude Code. העתק, התאם את הפרטים בסוגריים המרובעים, הדבק.

---

## 5 כללי זהב לפרומפט טוב

**1. הקשר לפני הוראה.**
לפני שאתה מבקש, ספר מי אתה, מה הפרויקט ומה הסטאק. Claude Code לא יודע את ההיסטוריה שלך אלא אם ספרת.

**2. תוצר, לא תהליך.**
אל תגיד "תכתוב קוד שעושה X". תגיד "אני רוצה שבסוף תהיה לי [תוצר מוגמר] שעושה [X]". Claude ידאג לתהליך.

**3. מגבלות ברורות.**
ציין מה אסור: "אל תשנה קבצים קיימים", "אל תוסיף ספריות חדשות", "צור Plan Mode לפני קוד". מגבלות מונעות הפתעות.

**4. דוגמת פלט.**
אם אפשר, הראה דוגמה קטנה של מה שאתה מצפה לקבל — JSON, מבנה תיקיות, עיצוב — גם אם לא מושלמת.

**5. בקש Plan Mode תמיד לפני בנייה גדולה.**
פרומפט שמתחיל ב-"לפני שאתה כותב קוד — שאל אם אין לך CLAUDE.md, הצג Plan ובקש אישור" חוסך לך שעות תיקונים.

## 1. דף נחיתה

**מתי משתמשים:**
כשרוצים לגייס לידים, לבדוק רעיון, לשווק מוצר חדש, או לפרסם אירוע. מדובר בעמוד אחד עצמאי עם CTA ברור. מתאים לכל מי שיש לו הצעת ערך ורוצה אתר מהיר בלי להשקיע בפיתוח מלא.

```
אתה בונה איתי דף נחיתה עצמאי בעברית ב-Next.js 14 (App Router).

### ההקשר
שם העסק/מוצר: [שם]
מה אני מוכר/מציע: [תיאור קצר — משפט אחד]
קהל יעד: [מי הלקוח האידיאלי]
CTA עיקרי: [מה אני רוצה שהמבקר יעשה — לדוגמה: "ישאיר פרטים", "יקנה", "ירשם"]

### לפני שאתה כותב קוד
1. בדוק אם קיים CLAUDE.md בפרויקט. אם כן — עקוב אחריו.
2. הצג Plan (מבנה סקשנים + קבצים שתיצור) ובקש אישורי לפני שמתחיל.

### דרישות תוכן
בנה את הדף עם הסקשנים הבאים (לפי סדר):
- Hero: כותרת ראשית שמבטאת תוצאה, תת-כותרת, כפתור CTA בולט
- בעיה/פתרון: 2–3 שורות שמתארות את הכאב ואיך אני פותר אותו
- יתרונות: 3 יתרונות בשפה של תוצאות (לא פיצ'רים)
- הוכחה חברתית: placeholder ל-3 המלצות (שם + ציטוט + תפקיד)
- CTA סופי: חזרה על הכפתור עם הצעה ברורה
- פוטר: קישורי מדיניות פרטיות, תנאי שימוש, נגישות (placeholders)

### דרישות טכניות
- שפה: עברית מלאה, dir="rtl" על html
- עיצוב: dark background, CSS variables ל-brand colors (אל תשתמש בצבעי Tailwind ישירים), פונטים: Polin או Assistant לעברית
- נגישות: WCAG 2.1 AA — alt לכל תמונה, aria-labels על buttons, ניגודיות תקנית
- טופס: שדות שם + טלפון + מייל, validation בסיסי, שלח ל-placeholder endpoint
- מובייל: responsive מלא, בדוק breakpoints 375px ו-768px
- אין URLs מומצאים. כל קישור חיצוני = placeholder [קישור]

### מה לא לעשות
- אל תוסיף ספריות חדשות בלי לשאול
- אל תיצור יותר מ-3 קבצים חדשים לפני שאישרתי את ה-Plan
- אל תכתוב עיצוב inline — רק CSS modules או Tailwind classes
```

```

2. חיפוש: שדה חיפוש לפי שם/מייל/טלפון
3. הוספת רשומה: טופס עם כל השדות, validation
4. עריכת רשומה: inline edit או modal
5. שינוי סטטוס: dropdown עם הסטטוסים הנ"ל
6. מחיקה: עם אישור ("האם אתה בטוח?")

### דרישות
- עברית מלאה, RTL
- responsive — עובד גם על טאבלט
- SQL: פרמטריזד תמיד, אין string concatenation
- אין hardcoded secrets — Supabase URL ו-key דרך env vars בלבד
- נגישות: keyboard navigation בטבלה ובטפסים

### מה לא לעשות
- אל תוסיף auth, roles, או permissions בשלב זה
- אל תיצור dashboard analytics — רק הטבלה
- אין over-engineering: אם פיצ'ר לא ברשימה — שאל לפני שמוסיף
```
